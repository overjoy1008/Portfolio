## 프로젝트 개요

TERENE는 실제 고객이 사용하는 숙박 예약 웹사이트 외주 개발 프로젝트입니다. Framer로 UI만 구현된 도안을 전달받아 예약, 결제, 멤버십, 쿠폰, 관리자 기능까지 포함한 전체 서비스를 혼자 설계하고 개발했습니다. 핵심 목표는 수동으로 처리되던 예약 프로세스를 실제 운영 가능한 자동화 서비스로 전환하는 것이었습니다.

## 사용 기술

- Framer
- React
- TypeScript
- Express
- PostgreSQL
- Render
- Toss Payments API
- SOLAPI

## Pain Point 및 목표 정립

- **기존 문제점:** 외주사가 전달한 결과물은 Framer로 UI만 구현된 상태였고, 예약 자동화, 결제, 멤버십, 쿠폰, 관리자 기능 같은 핵심 비즈니스 로직은 모두 비어 있었습니다. 또한 예약은 실제 고객 응대가 필요한 서비스였음에도 수동 처리에 의존해 완료까지 최대 24시간이 걸리고 있었습니다.
- **제약 조건:** Framer는 GitHub 연동이 되지 않고 코드 작성도 제한적이어서, 일반적인 React 프로젝트처럼 구조를 설계하기 어려웠습니다.
- **해결 목표:** 기존 도안을 유지한 채 예약, 결제, 알림, 회원 기능, 관리자 기능을 빠르게 구현하고, 실제 고객이 사용하는 예약 프로세스를 5초 내 자동화하는 것이 목표였습니다.

## 개발 구현 내용

- **Framer 기반 코드 컴포넌트 개발:** Framer에서 `.tsx` 형식의 코드 컴포넌트를 상속할 수 있다는 점을 활용해, 기존 디자인을 유지하면서 주요 UI 요소를 React 기반 코드 컴포넌트로 재구성했습니다.
- **상태 기반 프론트엔드 구현:** 서버에서 필요한 정보를 받아 State에 반영한 뒤 렌더링하는 구조로 토스 결제 위젯, 캘린더, 영수증, 쿠폰 생성 및 발급, 회원 및 예약 테이블 등을 구현했습니다.
- **예약·결제·알림 기능 통합:** 회원 및 비회원 예약 시스템, Toss Payments API 기반 결제, SOLAPI 기반 자동 알림톡을 연결해 예약 생성부터 확정, 후속 알림까지 이어지는 흐름을 구축했습니다.
- **운영 가능한 관리자 페이지 개발:** 코드를 모르는 관리자도 예약, 회원, 쿠폰 등 다양한 액션을 직접 처리할 수 있도록 관리자 페이지를 제작했습니다.
- **변동에 강한 데이터 구조 설계:** 빈번한 정책 변경에 대응하기 위해 마이그레이션을 최소화할 수 있는 정규화된 DB를 설계했고, 경우에 따라 DB 변경 없이 프론트엔드 코드만으로도 요구사항 변화에 대응했습니다.

## 트러블슈팅 및 해결 과정

- **문제 상황:** 개발 도중 예약 로딩 화면이 30초 이상 지연되는 병목 현상이 발생해 실제 예약 완료까지 사용자 경험이 크게 저하됐습니다.
- **원인 분석:** 관리자와 고객에게 발송되는 다수의 자동 알림톡이 여러 DB 작업과 함께 동기적으로 처리되면서, 예약 처리 전체 흐름을 블로킹하고 있었습니다.
- **해결 방안:** 모든 예약을 큐 기반으로 트래킹하고 처리하도록 설계했으며, 예약 처리 전용 API 서버와 자동 알림용 API 서버를 분리해 핵심 예약 흐름과 부가 작업을 분리했습니다.
- **결과:** 예약 프로세스를 5초 이내로 자동화했고, 이후 실사용 환경에서 발생한 기기·OS별 버그를 수정하며 안정적인 운영까지 이어갔습니다.

## 주요 성과

- **예약 자동화:** 24시간이 걸리던 예약 프로세스를 5초 이내로 자동화했습니다.
- **실서비스 운영:** 배포 후 70건 이상의 고객 예약을 처리하며 실사용 환경에서 운영과 개선을 지속했습니다.
- **회원 확보:** 15명의 멤버십 회원을 확보했습니다.
- **실무 역량 확장:** 정책 변경, 스키마 설계, 운영 이슈 대응, 관리자 UX 개선까지 경험하며 서비스 개발과 운영 전반의 감각을 익혔습니다.
