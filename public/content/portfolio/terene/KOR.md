# TERENE

**기간:** 2025.04 - 2026.01 (10개월)

**소속:** (주)바드건축사사무소 / 개발 외주

**직급:** 프리랜서

**역할:** 1인 외주 개발, Full-Stack 웹 개발

## 프로젝트 개요

TERENE는 숙박 예약 과정을 온라인에서 바로 처리할 수 있도록 구축한 숙박 예약 웹사이트입니다. 디자이너가 Framer로 전달한 도안을 바탕으로 예약, 결제, 알림, 관리자 기능까지 포함한 전체 서비스를 React/Express 기반으로 1인 풀스택 개발했습니다. 핵심 목표는 기존의 수동 예약 처리 흐름을 실제 운영 가능한 자동화 서비스로 전환하는 것이었습니다.

## 사용 기술

- Framer
- React
- TypeScript
- Express
- PostgreSQL
- Render
- Toss Payments API
- SOLAPI

## Pain Point 및 목표 정립

- **기존 문제점:** 예약 문의 접수부터 결제 확인, 예약 확정, 알림 발송까지 수동으로 처리되어 예약 완료까지 최대 24시간이 소요되었습니다.
- **해결 목표:** 회원/비회원 모두 사용할 수 있는 예약 흐름을 구축하고, 결제와 알림을 API로 연동해 예약 처리 시간을 수초 단위로 줄이는 것이 목표였습니다.

## 개발 구현 내용

- **예약 파이프라인 구축:** 회원/비회원 예약 시스템을 설계하고 Toss Payments API 기반 결제, SOLAPI 기반 카카오 알림톡 발송을 연결해 예약 생성부터 확정까지 이어지는 흐름을 자동화했습니다.
- **회원 및 관리자 기능 개발:** 인증, 예약 조회, 회원 정보, 쿠폰, 마일리지 현황을 제공하는 회원 페이지와 운영용 관리자 페이지를 구현했습니다.
- **RESTful 서버 API 설계:** DB 접근용 API, 자동 알림용 API, 토스 결제용 API를 역할별로 분리해 Express 서버에서 RESTful하게 설계하고 Render에 배포했습니다.
- **운영 대응 및 유지보수:** 예약 변경, 취소, 결제 상태 반영, 예외 케이스 대응과 버그 수정을 지속적으로 수행하며 실제 서비스 운영 경험을 쌓았습니다.

## 트러블슈팅 및 해결 과정

- **문제 상황:** 숙박 예약 서비스는 회원/비회원 흐름, 결제 성공/실패, 예약 변경 및 취소처럼 상태 분기가 많아 작은 누락도 운영 이슈로 이어질 수 있었습니다.
- **원인 분석:** 예약 상태, 결제 상태, 알림 발송 시점이 서로 다른 이벤트로 움직이기 때문에 하나의 로직에 과도하게 결합되면 예외 상황에서 데이터 불일치가 발생할 가능성이 있었습니다.
- **해결 방안:** 예약, 결제, 알림 API를 역할별로 분리하고 각 단계의 상태 전이를 기준으로 처리 순서를 정리해 예외 케이스에도 안정적으로 후속 작업이 이어지도록 구성했습니다.
- **결과:** 실제 운영 환경에서 예약 자동화가 안정적으로 동작했고, 추가 기능 개발과 버그 수정까지 포함한 운영 관리 경험으로 이어졌습니다.

## 주요 성과

- **예약 자동화:** 예약 처리 시간을 24시간 수동 처리에서 5초 내 자동화로 단축했습니다.
- **서비스 운영 성과:** 배포 후 누적 70건의 숙박 예약을 완료했고, 15명의 멤버십 회원을 확보했습니다.
- **실서비스 공개:** 실제 운영 사이트를 배포해 서비스 인수인계까지 완료했습니다. [https://terene.kr/](https://terene.kr/)
