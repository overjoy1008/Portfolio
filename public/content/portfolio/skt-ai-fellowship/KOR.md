**기간:** 2024.06 - 2024.10 (5개월)

**소속:** SK Telecom / AI Fellowship 6기

**직급:** AI Fellow (VR/AI 개발)

**역할:** 학부생 3인 프로젝트 진행, VR 인프라 구축 및 AI 기능 구현

## 프로젝트 개요

이 프로젝트는 Meta Quest 3 기반 VR 환경에서 동작하는 AI 학습 어시스턴트를 구축하는 과제였습니다. 팀 내에 Unity를 능숙하게 다루는 개발자가 없고, 팀원의 갑작스러운 해외 일정으로 대면 테스트가 어려운 상황에서도 프로젝트를 완주해야 했습니다. 저는 필요한 Use Case를 분석해 개발 영역을 VR·Unity, 2D→3D, LLM으로 분할하고, 그중 가장 난이도가 높았던 VR·Unity 개발을 맡아 전체 시스템 통합을 주도했습니다.

## Pain Point 및 목표 정립

- **팀 협업 문제:** 팀은 학부생 3인으로 구성되어 있었고, 팀원 한 명의 해외 일정으로 인해 VR 기기 컴파일과 테스팅을 함께 수행하기 어려운 환경이었습니다.
- **기술 역량 문제:** 팀 내에 Unity를 능숙하게 다루는 개발자가 없어 VR 주제를 수행하는 것 자체가 높은 진입장벽이었습니다.
- **해결 목표:** 제한된 대면 협업 환경에서도 각자가 독립적으로 개발할 수 있는 구조를 만들고, 최종적으로 VR 환경에서 음성 질문, 3D 생성, AI 응답이 통합된 학습 어시스턴트를 구현하는 것이 목표였습니다.

## 개발 구현 내용

- **개발 영역 분할:** 필요한 Use Case를 분석해 VR·Unity, 2D→3D, LLM 영역으로 업무를 분할하고 각 팀원이 독립적으로 개발 가능한 구조를 만들었습니다.
- **VR·Unity 인프라 구축:** 한 달간 Unity를 독학하며 Meta Quest 3 환경을 동기화하고, 학습용 VR 씬과 상호작용 기능을 직접 구현했습니다.
- **서버 API 통합:** 각 팀원이 개발한 기능을 서버 API로 통합해 Unity와의 실시간 연동에 성공했고, 주간 비대면 협업만으로도 합류 시점에 즉각적인 시스템 통합이 가능하도록 모듈화했습니다.
- **AI 파이프라인 구현:** 음성 질문 처리, 의도 분류, 2D 이미지의 3D 변환, LLM 응답 생성을 각각 연결해 하나의 VR 학습 경험으로 통합했습니다.
- **피보팅 대응:** 회사에서 중간에 주제 피보팅을 요구했을 때도, 모듈화된 구조 덕분에 전체 로직과 서버 기능은 유지한 채 일부 입력 데이터만 바꿔 빠르게 전환할 수 있었습니다.

## 트러블슈팅 및 해결 과정

- **문제 상황:** 팀 내 Unity 경험 부족, 제한된 대면 테스트, 중도 피보팅 요구까지 겹치며 프로젝트 완주 자체가 쉽지 않은 상황이었습니다.
- **원인 분석:** 기술적 난이도뿐 아니라 팀의 협업 구조가 정리되지 않으면 VR 기기 의존성이 큰 프로젝트에서 통합이 계속 늦어질 가능성이 컸습니다.
- **해결 방안:** 먼저 Use Case 기준으로 기능을 명확히 분할해 병렬 개발이 가능하게 만들고, 제가 VR·Unity와 시스템 통합을 책임지며 각 기능을 API 단위로 연결했습니다. 또한 명확한 의도 분류 체계를 설계해 적은 데이터로도 성능을 확보할 수 있도록 했습니다.
- **결과:** 적은 데이터에서도 STT 기반 의도 분류 정확도 97%와 약 2초의 응답 시간을 달성했고, 최종 시연까지 성공적으로 마쳤습니다.

## 주요 성과

- **프로젝트 완주:** 악조건 속에서도 모듈화된 분업과 통합 구조를 통해 프로젝트를 성공적으로 완주했습니다.
- **성능 달성:** STT 의도 분류 정확도 97%, 응답 시간 약 2초를 달성했습니다.
- **유연한 구조 설계:** 피보팅 상황에서도 전체 시스템을 유지한 채 일부 입력만 교체해 빠르게 전환할 수 있는 구조를 만들었습니다.
