# VR 기반 AI 학습 어시스턴트 (SKT AI Fellowship)

**기간:** 2024.06 - 2024.10 (5개월)

**소속:** SK Telecom / AI Fellowship 6기

**직급:** AI Fellow (VR/AI 개발)

**역할:** 학부생 3인 프로젝트 진행, VR 인프라 구축 및 AI 기능 구현

## 프로젝트 개요

이 프로젝트는 Meta Quest 3 기반 VR 환경에서 동작하는 AI 학습 어시스턴트를 구축하는 과제였습니다. Unity로 학습용 VR 인프라를 만들고, 서버 API 및 AI 파이프라인을 연결해 음성 질문 응답과 PDF 이미지 기반 3D 모델 생성 기능을 제공했습니다. 목표는 VR 환경 안에서 학습 콘텐츠 탐색, 음성 질의응답, 시각 자료의 3D 변환까지 자연스럽게 연결되는 몰입형 학습 경험을 만드는 것이었습니다.

- Unity

- LLM

- FastAPI
- AWS S3
- AWS EC2

## Pain Point 및 목표 정립

- **기존 문제점:** VR 학습 환경에서는 단순한 콘텐츠 열람만으로는 상호작용성이 부족했고, 텍스트나 2D 이미지 중심 자료를 몰입형 학습 경험으로 확장하기 어려웠습니다.
- **해결 목표:** VR 내 이동, 버튼 상호작용, 음성 질문, PDF eBook 열람, 이미지의 3D 변환까지 하나의 흐름으로 연결해 실질적인 AI 학습 어시스턴트 경험을 구현하는 것이 목표였습니다.

## 개발 구현 내용

- **VR 인프라 구축:** Unity에서 VR 학습용 씬을 개발하고 이동, 버튼 상호작용, 음성 질문, PDF eBook 기능을 제공하는 기본 학습 환경을 구성했습니다.
- **Image-to-3D 파이프라인:** 업로드한 PDF 내 학습 이미지를 서버에서 U2-Net으로 전처리한 뒤 Meshy.ai API로 3D 모델을 생성하고, 이를 Unity 씬에 불러오는 파이프라인을 구축했습니다.
- **음성 질문 기능 구현:** Meta Quest 3에서 입력된 음성을 Wit.ai 기반 STT와 의도 분류로 처리한 뒤, 질문 목적에 맞는 AI 응답이 이어지도록 서버 파이프라인을 개발했습니다.
- **할루시네이션 완화:** STT 변환 과정에서 핵심 키워드를 추출하고 위키 기반 문단을 함께 참고해 LLM이 답변하도록 설계했으며, 참고한 링크를 Unity 씬 안에서 확인할 수 있게 했습니다.

## 트러블슈팅 및 해결 과정

- **문제 상황:** VR 기기에서의 음성 입력은 지연 시간이 길거나 의도 분류가 부정확하면 사용자 경험이 크게 저하되고, LLM 답변은 근거 없는 설명으로 이어질 위험이 있었습니다.
- **원인 분석:** 음성 질문은 STT 오인식, 의도 분류 오류, 외부 지식 부족이 연쇄적으로 누적되기 쉬워 단일 모델만으로는 안정적인 응답 품질을 보장하기 어려웠습니다.
- **해결 방안:** 질문 데이터셋을 자체 제작해 의도 분류를 고도화하고, 핵심 키워드 추출과 위키 크롤링 문단을 결합해 LLM 답변의 근거를 보강했습니다.
- **결과:** STT 및 의도 분류에서 97% confidence와 1~2초 latency를 달성했고, 학습 이미지의 3D 모델 변환까지 포함한 확장 가능한 VR-AI 파이프라인을 완성했습니다.

## 주요 성과

- **음성 처리 성능:** STT와 의도 분류에서 97% confidence 및 1~2초 latency를 달성했습니다.
- **모듈 확장성 확보:** PDF 단면 이미지를 360도 3D 모델로 변환하는 모듈을 구축해 다양한 Use Case로 확장 가능한 기반을 마련했습니다.
