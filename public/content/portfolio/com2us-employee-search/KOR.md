**소속:** (주)컴투스 / ECO실 (클라이언트 개발)

**역할:** 1인 프로젝트 진행, LLM·RAG 기반 챗봇 개발

## 프로젝트 개요

이 프로젝트는 사내에서 특정 업무의 담당자를 자연어 질문만으로 빠르게 찾을 수 있도록 만든 직원 검색용 AI 챗봇입니다. 방대한 인력 규모에 비해 각 직원의 세부 업무 정보가 파편화되어 있어 적절한 담당자를 찾는 데 과도한 시간과 절차가 소요되는 문제를 해결하고자 했습니다. 이를 위해 개발자들의 주간보고서를 기반으로 LLM·RAG 기반 검색 시스템과 Self-Updating 파이프라인을 구축했습니다.

## Pain Point 및 목표 정립

- **기존 문제점:** 사내 인력 규모는 크지만 각 직원이 실제로 어떤 프로젝트에서 어떤 세부 업무를 맡고 있는지에 대한 정보가 여러 문서와 사람 사이에 파편화되어 있어, 담당자를 찾는 데 많은 시간과 절차가 필요했습니다.
- **데이터 품질 문제:** 주간보고서를 기반으로 RAG를 구축했지만, 사람마다 작성 형식이 달라 임베딩 과정에서 데이터 불균형이 발생했고 검색 정확도가 낮았습니다.
- **도메인 특화 문제:** 게임 개발 조직 특성상 유사한 용어가 많아, 임베딩 모델이 서로 다른 프로젝트와 업무를 명확히 구분하는 데 어려움이 있었습니다.
- **해결 목표:** 특정 업무에 대한 자연어 질문만으로 5초 내 담당자를 찾아주는 AI 챗봇을 만들고, 매주 최신 데이터를 반영해 정확도를 지속적으로 개선하는 검색 파이프라인을 구축하는 것이 목표였습니다.

## 개발 구현 내용

- **직원 검색 챗봇 개발:** 특정 업무나 프로젝트를 입력하면 가장 관련도 높은 담당자를 추천하는 사내 검색용 챗봇을 구현했습니다.
- **보고서 정규화 자동화:** 작성 형식이 제각각인 주간보고서를 LLM으로 요약하고 JSON 형태로 정규화하는 파이프라인을 구축해 전처리를 자동화했습니다.
- **Self-Updating RAG 구축:** 정규화된 보고서를 벡터화해 임베딩 공간에 적재하고, 주간 단위로 최신 데이터가 누적 반영되도록 Self-Updating RAG 구조를 설계했습니다.
- **임베딩 모델 파인튜닝:** 용어 간 구분력이 낮은 문제를 해결하기 위해 10년치 퇴사자 기반 데이터셋을 구축하고 임베딩 모델을 파인튜닝했습니다.
- **데이터 선순환 구조 제안:** 매주 주간보고서를 구체적으로 작성한 개발자를 선별하는 사내 포상 제도를 제안해, 고품질 데이터가 지속적으로 유입되는 운영 구조를 만드는 데 기여했습니다.

## 트러블슈팅 및 해결 과정

- **문제 상황:** 초기 챗봇은 주간보고서를 그대로 임베딩했기 때문에 문서 형식 편차가 크고, 유사한 게임 개발 용어가 많아 실제 담당자를 정확히 찾지 못하는 경우가 잦았습니다.
- **원인 분석:** 검색 품질 저하의 핵심 원인은 모델 자체보다 입력 데이터의 불균형과 도메인 용어에 대한 낮은 구분력이었습니다.
- **해결 방안:** 주간보고서를 JSON 형태로 정규화 및 요약하는 자동화 파이프라인을 만들고, 10년치 데이터셋으로 임베딩 모델을 파인튜닝해 업무와 프로젝트 단위의 의미 구분력을 높였습니다.
- **결과:** 코사인 유사도와 Recall@10이 크게 향상되었고, 실제 사내에 배포되어 제가 속한 부서뿐 아니라 다른 부서에서도 활용되기 시작했습니다.

## 주요 성과

- **검색 품질 향상:** 코사인 유사도를 22.7% 향상시켜 0.869를 달성했습니다.
- **랭킹 성능 개선:** Recall@10을 4.5배 수준으로 향상시켰고, ndcg@10도 함께 개선했습니다.
- **운영 확산:** 챗봇을 실제 사내에 배포해 근무 부서 및 다른 부서에서도 활용되기 시작했습니다.
- **데이터 관점의 인사이트 확보:** AI 성능 개선의 핵심이 모델 변경만이 아니라 데이터 이해, 데이터 품질 개선, 지속적인 실험에 있다는 점을 체감했습니다.
