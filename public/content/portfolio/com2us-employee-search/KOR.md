# 컴투스 사내 직원 검색용 Self-Updating RAG

**기간:** 2024.12 - 2025.02 (3개월)

**소속:** (주)컴투스 / ECO실 (클라이언트 개발)

**직급:** 인턴

**역할:** 1인 프로젝트 진행, LLM·RAG 기반 챗봇 개발

## 프로젝트 개요

이 프로젝트는 특정 프로젝트나 업무의 최신 담당자를 더 빠르게 찾을 수 있도록 만든 사내 직원 검색용 챗봇입니다. 기존의 템플릿형 사내 챗봇을 LLM과 RAG 기반 시스템으로 고도화해, 사용자의 자연어 질문에 맞춰 실제로 필요한 담당자 정보를 추천하도록 설계했습니다. 또한 주간보고서를 자동 요약하고 벡터 DB를 최신 상태로 유지하는 Self-Updating RAG 파이프라인을 함께 구축했습니다.

## Pain Point 및 목표 정립

- **기존 문제점:** 사내 직원 DB에는 직위 정보는 있었지만, 각 개발자가 어떤 프로젝트에서 어떤 업무를 맡았는지에 대한 최신 맥락 정보가 부족해 담당자를 찾는 과정이 비효율적이었습니다.
- **해결 목표:** 개발자 주간보고서를 활용해 업무 맥락이 반영된 최신 검색 인덱스를 만들고, 자연어 질문만으로도 적합한 담당자를 추천할 수 있는 검색형 챗봇을 구현하는 것이 목표였습니다.

## 개발 구현 내용

- **직원 검색 챗봇 개발:** 특정 프로젝트나 업무의 최신 담당자를 찾는 과정을 단순화하기 위해 사내 네이버웍스 환경에서 활용 가능한 직원 검색용 챗봇을 개발했습니다.
- **기존 챗봇 고도화:** 정해진 템플릿에 맞춘 응답 방식에서 벗어나, LLM과 RAG를 통해 질문 의도에 맞는 맞춤형 정보를 제공하도록 개선했습니다.
- **Self-Summarizing·Updating RAG 파이프라인:** Qwen2.5-32B-Instruct-GPTQ-Int4가 매주 클라이언트 개발자의 주간보고서를 요약하고 JSON으로 정규화한 뒤 벡터화해 Milvus에 최신 정보를 적재하는 자동화 서버를 구축했습니다.
- **임베딩 모델 파인튜닝:** 10년치 주간보고서를 기반으로 multilingual-e5-large를 파인튜닝해 사내 도메인 용어와 프로젝트 맥락을 더 잘 반영하도록 검색 품질을 개선했습니다.

## 트러블슈팅 및 해결 과정

- **문제 상황:** 단순 직원 DB 검색만으로는 실제 업무 담당자와 검색 결과 사이의 맥락 차이가 커서, 최신 프로젝트 담당자를 정확히 찾기 어려웠습니다.
- **원인 분석:** 직원 DB는 정적인 인사 정보 중심이었고, 실제 업무 이력과 주간 단위 역할 변화는 보고서에만 흩어져 있어 검색 인덱스가 최신성을 잃기 쉬웠습니다.
- **해결 방안:** 주간보고서를 LLM으로 자동 요약 및 정규화한 뒤 벡터 DB를 주기적으로 갱신하는 self-updating 파이프라인을 설계했고, 임베딩 모델을 파인튜닝해 사내 업무 용어에 대한 검색 정밀도를 높였습니다.
- **결과:** 최신 담당자 탐색 정확도가 개선되었고, 배포 이후 컴투스 클라이언트 부서뿐 아니라 다른 부서들로도 활용 범위가 확장되었습니다.

## 주요 성과

- **검색 정확도 개선:** 파인튜닝한 임베딩 모델로 검색 정확도(Cosine Similarity) 0.869를 달성했습니다.
- **랭킹 성능 향상:** recall@10은 0.04에서 0.18로, ndcg@10은 0.02에서 0.11로 상승했습니다.
- **사내 확장성 확보:** 배포 후 컴투스 클라이언트 부서 및 다른 부서에도 활용되며 사내 AI 서비스 확대에 기여했습니다.
